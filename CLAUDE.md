# CVbuilder

A web-based CV/resume generator built with Flask and vanilla JavaScript. Reads CV data from `content.json` and renders it via Jinja2 templates with dynamic filtering, layout switching, color palette selection, and PDF export.

## Tech Stack

- **Backend:** Python 3.9+, Flask, Waitress (WSGI server)
- **Frontend:** Jinja2 templates, jQuery, jsPDF, html2canvas, html2pdf.js, FontAwesome
- **Dependency management:** pip + npm (npm postinstall copies libs to `static/javascript/libs/`)

## Project Structure

```
backend/        # Flask app
  __main__.py   # Entry point (arg parser)
  router.py     # Routes: GET /, GET /health, POST /filter
  experience_handler.py  # Experience filtering and HTML rendering
  utils.py      # JSON content loader
templates/      # Jinja2 templates (base, index, nav_menu, side_panel)
static/
  javascript/   # JS modules + libs/ (minified dependencies)
  stylesheets/  # CSS files with CSS custom properties for theming
  fontawesome/
content.json    # All CV data (the only file to edit for CV content)
```

## Setup

```bash
pip install -r requirements.txt
npm install   # also copies JS libs to static/javascript/libs/
```

## Running

```bash
python backend              # port 3000, production (Waitress)
python backend --dev_mode   # Flask debug server
python backend --port 8080  # custom port
```

Docker: exposes port 8080.

## Key Conventions

- **CV content lives entirely in `content.json`** — no code changes needed for content updates.
- **No database or environment variables** — fully self-contained.
- **Frontend filtering is AJAX-based:** `filter.js` POSTs to `/filter`, which returns HTML rendered by `ExperienceHandler.cast_to_html()`.
- **Theming is CSS-variable-based** — color palettes are switched by JavaScript updating CSS custom properties on the root element.
- **JS modules use ES6 imports** — `index.html` loads `filter.js` as `type="module"`, which imports other modules.
- **Code formatting:** Python uses Black (`black .`).
- Static JS libraries are committed to `static/javascript/libs/` (generated by `npm install`), so `node_modules/` is gitignored.
